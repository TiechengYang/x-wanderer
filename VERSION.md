# X Wanderer - 版本快照

**版本标识**: v0.3.0-alpha (123 完整版)  
**保存时间**: 2026 年（基于当前对话状态）

## 核心特性（已实现）

- 完全自主的 LangGraph 决策循环（平衡型）
- 重度记忆系统（Chroma + SQLite）
- 人物 & 话题长期画像系统（支持主动维护）
- 主动回访重要画像能力（Wander + Engage 联动）
- LLM 驱动的高质量反思 + 重要关系总结
- 长期目标管理 + 进度追踪（含状态评估）
- 后台智能记忆维护定时任务
- 人类实时干预支持（文件 + 命令行）
- X API 基础集成（带限流保护和重试机制）
- 干运行模式（`main_dryrun.py`）

## 当前能力水平

- **记忆与认知**：★★★☆☆（已有画像积累 + 主动维护）
- **自主决策**：★★★☆☆（平衡型 + 目标联动）
- **关系维护**：★★★☆☆（支持主动回访）
- **目标管理**：★★★☆☆（有进度记录和状态评估）
- **工程化**：★★★☆☆（有 setup 脚本 + 干运行 + 完整性测试）

## 主要限制

- 行为仍较为原型化，策略深度不足
- X API 调用成本与风控需自行控制
- 尚未实现真正的多轮关系演化与复杂规划
- 画像使用仍处于早期阶段

## 如何运行

1. 推荐使用一键脚本：
   ```bash
   bash setup.sh
   ```

2. 配置 `.env` 后运行：
   ```bash
   python3 main.py
   ```

3. 测试模式（无需真实 API Key）：
   ```bash
   python3 main_dryrun.py
   ```

## 文件清单（关键）

- `main.py`               → 正式运行入口
- `main_dryrun.py`        → 干运行测试入口
- `setup.sh`              → 一键安装脚本
- `test_integrity.py`     → 完整性测试
- `agent/graph.py`        → 核心决策图
- `agent/memory/`         → 重度记忆 + 画像系统
- `utils/scheduler.py`    → 后台维护任务
