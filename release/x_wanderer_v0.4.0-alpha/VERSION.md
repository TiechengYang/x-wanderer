# X Wanderer - 版本快照

**版本标识**: v0.4.0-alpha (ABC 结构化铁律强化版)  
**保存时间**: 2026 年（基于当前对话状态）

## 核心特性（已实现）

- 完全自主的 LangGraph 决策循环（平衡型 + 激进关系维护模式）
- 重度记忆系统（Chroma + SQLite + 结构化关系总结）
- 人物 & 话题长期画像系统（支持主动维护 + 重要性/回访时间追踪）
- **结构化铁律多轮回访战役**（v0.4.0 新）：analyze_people 后自动注入高优先级目标，supervisor 在 pre-LLM 阶段直接强制连续 engage，显式 `current_revisit_campaign` 状态
- 关系图谱 + 桥梁人物 + 聚类自动提取（结构化 JSON 版）
- 分析后强目标改写 + 子目标注入（analyze_people 直接突变 current_goal / sub_goals / goal_status）
- 每次回访后自动写关系总结（闭环：行动 → 记忆 → 下次分析更精准）
- 调度器激进种子注入（直接把高价值 stale 画像推进回访队列）
- LLM 驱动的高质量反思 + 重要关系总结
- 长期目标管理 + 进度追踪（含状态评估 + 分析驱动的目标重定向）
- 后台智能记忆维护定时任务（支持直接目标种子）
- 人类实时干预支持（文件 + 命令行，支持 /analyze_people 强制触发）
- X API 基础集成（带限流保护和重试机制）
- 干运行模式（`main_dryrun.py`，带战役可视化仪表盘）

## 当前能力水平

- **记忆与认知**：★★★★☆（结构化关系图谱 + 闭环总结）
- **自主决策**：★★★★☆（结构化铁律 + 显式战役状态，几乎不可能逃脱多轮回访）
- **关系维护**：★★★★☆（真正的多轮持续战役 + 分析后立即驱动）
- **目标管理**：★★★★☆（分析结果可强力改写目标和子目标）
- **工程化**：★★★★☆（干运行仪表盘可视化战役、完整性测试、发布脚本）

## 主要限制

- 真实 X 环境仍需谨慎风控（白名单 + AI 身份声明已强制）
- 多轮战役的“结束条件”目前主要靠连续行动计数，仍可进一步精细化
- 尚未对接 Reddit / 微信公众号等其他平台（Stage-1 聚焦 X）

## v0.4.0-alpha 重大更新（ABC 激进强化）

- Supervisor 引入**结构化铁律早返回机制**：只要存在 active_revisit_targets，决策阶段直接强制 engage，完全绕过 LLM（核心强化）
- 新增显式 `current_revisit_campaign` 状态机（target + streak），让多轮回访从“概率行为”变成“一等公民”
- analyze_people 后直接重写目标系统 + 灌入子目标 + 拉低进度分，强迫系统重视关系维护
- Engage 每轮回访自动写入结构化关系总结，实现“打 → 记 → 分析 → 更准的打”闭环
- Scheduler 可直接种子高价值回访目标（比单纯建议 analyze 更激进）
- 干运行仪表盘显著增强，可直观看到战役启动、streak 增长、目标被分析改写全过程
- 修复了早期 memory_manager 截断导致的“无反应”问题

此版本标志着 Agent 从“会主动回访”进化到“**会强迫自己持续维护高价值关系**”。

## 如何运行

1. 推荐使用一键脚本：
   ```bash
   bash setup.sh
   ```

2. 配置 `.env` 后运行：
   ```bash
   python3 main.py
   ```

3. 测试模式（无需真实 API Key）：
   ```bash
   python3 main_dryrun.py
   ```

## 文件清单（关键）

- `main.py`               → 正式运行入口
- `main_dryrun.py`        → 干运行测试入口
- `setup.sh`              → 一键安装脚本
- `test_integrity.py`     → 完整性测试
- `agent/graph.py`        → 核心决策图
- `agent/memory/`         → 重度记忆 + 画像系统
- `utils/scheduler.py`    → 后台维护任务
