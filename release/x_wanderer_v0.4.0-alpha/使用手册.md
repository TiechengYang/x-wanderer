# X Wanderer 使用手册

本手册帮助你更好地理解和使用这个数字漫游者 Agent。

## 1. 核心概念

### 1.1 长期画像（Profiles）
Agent 会长期积累对**人物**和**话题**的印象，称为“画像”。

- 人物画像：记录某个 Twitter/X 用户的特征、兴趣、互动历史。
- 话题画像：记录某个主题（如 AI 安全、数字游民文化）的关键洞见。

这些画像会影响 Agent 的决策、反思和互动策略。

### 1.2 主动回访（Proactive Revisit）
Agent 不是被动等待内容，而是会**主动**去关注它认为重要的画像。

- 通过 `suggest_profiles_to_revisit()` 机制，定时任务会推荐需要回访的对象。
- Wander 和 Engage 节点会优先处理这些回访建议。

### 1.3 目标管理系统
Agent 拥有长期目标，并会追踪进度：

- `current_goal`：当前主要目标
- `sub_goals`：子目标
- `goal_progress_notes`：目标进展记录
- `goal_status`：目标状态（active / progressing / stagnant / needs_revision）

Supervisor 会定期评估目标状态，并在必要时调整。

### 1.4 重度记忆
记忆分为多层：
- 短期记忆（当前上下文）
- 情景记忆（具体事件）
- 语义记忆（向量检索）
- 反思记忆（最高价值）
- 画像记忆（长期关系与主题认知）

## 2. 日常使用

### 2.1 启动 Agent

推荐方式：
```bash
bash setup.sh          # 一键准备环境
python3 main.py        # 正式运行
```

### 2.2 人类干预方式

支持两种实时干预方式（无需重启）：

**方式 A：文件干预（推荐用于复杂指令）**
```bash
echo "/goal 重点关注AI伦理和长期主义相关讨论" > commands/human_input.txt
```

**方式 B：命令行干预**
运行 `main.py` 后，直接在终端输入指令，例如：
- `/goal 把重点放在技术哲学方向`
- `/reflect`
- `/pause`
- `/status`

### 2.3 查看当前状态

干运行模式非常适合观察内部状态：
```bash
python3 main_dryrun.py
```

它会每轮打印关键决策、目标、反思等信息。

## 3. 关键模块说明

| 模块                  | 作用                              | 是否推荐手动修改 |
|-----------------------|-----------------------------------|------------------|
| `agent/nodes/supervisor.py` | 核心大脑，决策 + 目标管理 + 画像联动 | 谨慎修改 Prompt |
| `agent/nodes/reflect.py`    | 反思 + 关系总结                   | 可调整反思深度 |
| `agent/nodes/engage.py`     | 主动互动                          | 可调整回复风格 |
| `agent/memory/memory_manager.py` | 记忆与画像统一管理            | 高级用户可扩展 |
| `utils/scheduler.py`        | 后台定时记忆维护                  | 可调整执行频率 |
| `main_dryrun.py`            | 无 API 依赖的测试模式             | 推荐用于调试 |

## 4. 最佳实践建议

1. **初期建议用干运行模式多跑几天**，观察 Agent 的决策偏好和画像积累情况。
2. **不要频繁修改目标**。让 Agent 有足够时间围绕一个目标积累画像和记忆。
3. **善用人类干预**。当你发现 Agent 偏离方向时，及时用 `/goal` 进行纠偏。
4. **定期查看重要画像**。可以通过修改 Scheduler 或添加打印逻辑来观察当前最重要的画像。
5. **控制 X API 用量**。真实运行时注意频率，避免触发限制。

## 5. 常见问题

**Q: Agent 一直不 engage，怎么办？**  
A: 可以在 Supervisor Prompt 中降低 engage 的门槛，或直接通过人类指令强制它去和某个用户互动。

**Q: 画像积累太慢怎么办？**  
A: 可以降低 `suggest_profiles_to_revisit` 的门槛，或增加定时任务的执行频率。

**Q: 想让 Agent 更激进还是更保守？**  
A: 修改 `agent/nodes/supervisor.py` 中的 `SUPERVISOR_SYSTEM_PROMPT`，调整“平衡型”的描述。
